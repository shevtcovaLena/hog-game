# 🎮 HOG-App (Hidden Objects Game) — Состояние проекта

## 📋 Описание

**HOG-App** — браузерная игра поиска предметов (Hidden Objects Game) на **PixiJS 8** (TypeScript, Vite).  
Игрок должен найти и собрать все предметы на фоновом изображении. При завершении уровня показывается финальный экран с кнопкой перезапуска.

```

```

## 🔧 Система координат

### Принцип
- **Центр мира** = левый верхний угол фона (0, 0)
- **Фон масштабируется** через `worldContent.scale` для адаптации под размер окна
- **Предметы позиционируются** в базовых координатах фона и масштабируются вместе с контейнером
- **Viewport** остаётся камерой; сам не масштабируется, отслеживает границы через `worldWidth/worldHeight`

### Масштабирование

```
scale = max(screenWidth / bgWidth, screenHeight / bgHeight)
worldContent.scale.set(scale)
viewport.worldWidth = bgWidth × scale
viewport.worldHeight = bgHeight × scale
```

Это обеспечивает **адаптивность**: при ресайзе окна фон и все предметы одновременно пересчитывают размер и остаются в допустимой области.

## 🎯 Функциональность

### ✅ Реализовано

- **Загрузка ассетов**: фон и спрайты из атласа (Spritesheet)
- **Адаптивный фон**: масштабируется под размер окна без искажения aspect ratio
- **Спавн предметов**: случайное расположение в пределах фона с паддингом
- **Сбор предметов**: клик/тап по предмету — исчезает с анимацией
- **Завершение уровня**: при сборе всех предметов показывается финальный экран
- **Перезапуск игры**: без перезагрузки страницы (сброс состояния «на лету»)
- **Управление камерой**:
  - **Drag** (мышь/тач один палец) — перетаскивание мира
  - **Pinch** (два пальца) — zoom в/из на мобильных
  - **Wheel/Trackpad** (десктоп) — zoom через колёсико
  - **Clamp** — предотвращение выхода за границы, динамический min-zoom
  - **Decelerate** — инерционное замедление при отпускании
- **Адаптивность**: корректное поведение при ресайзе окна

### 📦 Используемые пакеты

- **pixi.js** (v8.x) — 2D-рендеринг
- **pixi-viewport** — управление камерой (drag, pinch, zoom, clamp, decelerate)
- **vite** — сборщик проекта
- **typescript** — типизация

## 📝 Недавние изменения

### Рефакторинг архитектуры (финальная версия)

1. **Введён промежуточный контейнер `worldContent`**
   - Фон и предметы находятся внутри одного контейнера
   - Единое масштабирование при ресайзе
   - Viewport остаётся камерой без внутреннего масштабирования

2. **Viewport из pixi-viewport**
   - Замена самописной логики drag/zoom на проверенное решение
   - Плавные жесты на мобильных устройствах
   - Встроенные плагины: drag, pinch, wheel, decelerate, clamp, clampZoom

3. **Логика позиционирования предметов**
   - Исправлен расчёт координат относительно центра фона
   - Предметы всегда появляются внутри видимой области с паддингом

4. **Централизованное управление игровым состоянием**
   - `Game.endGame()` — единая точка входа для завершения
   - `Game.restart()` — сброс состояния
   - `Game.spawnItems()` — создание предметов заново

## 🎮 Как играть

1. **Загрузка**: при старте игра инициализирует viewport, фон и предметы
2. **Поиск**: нужно найти и кликнуть/тапнуть на все предметы
3. **Управление сценой**:
   - 🖱️ Перетащить мышью или одним пальцем
   - 🔍 Zoom колёсиком (десктоп) или двумя пальцами (мобиль)
4. **Завершение**: собрав все предметы, появляется финальный экран
5. **Перезапуск**: кнопка «Перезапустить» на финальном экране

## 💡 Ключевые решения по архитектуре

1. **Единая система координат**: центр мира в левом верхнем углу фона (без якоря)
2. **Масштабирование через контейнер**: `worldContent` масштабируется, предметы остаются прикреплены
3. **Viewport как камера**: управляет pan/zoom/clamp, но не участвует в масштабировании содержимого
4. **Callback-базированная архитектура**: `Game` управляет потоком через методы `endGame()` и `restart()`
5. **Адаптивность по дизайну**: все размеры и позиции вычисляются динамически при ресайзе
